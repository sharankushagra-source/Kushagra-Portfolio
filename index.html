Case study · Product

Factile Customize

Factile is a live classroom quiz-game platform with 2M+ players. I rebuilt game customization — themes, media uploads, mascot selection — the surface a teacher uses to make a game their own.

Role
Lead product designer
Platform
Desktop-first web app
Audience
Older, non-technical educators
Focus
Customization & accessibility
Why revamp

Customization sat inside overloaded settings carrying three layers of debt — inconsistent UI, unclear interactions, and a product model that didn't match how teachers think. First-timers couldn't predict what a control would do.

What I did

Untangled it into one coherent, self-explaining flow: clearer states, predictable interactions, and a WCAG AA/AAA pass validated against an accessibility inventory comparison.

Flagship · Design system

Accessible Foundations

The design system behind Factile. The legacy app was built straight in code by developers, so visual and interaction consistency drifted screen to screen — the foundation the product never had.

Role
Design-system designer
Platform
Desktop-first web app
Rollout
Shipped step by step
Inside
Interactive product demo
Why it was needed

Factile grew feature by feature with no shared foundation. Colour, spacing, components and states diverged across the app, and every new build re-litigated the basics — a real consistency problem.

What I did

Reverse-engineered the product into tokens, components and patterns — WCAG AA/AAA by default — and rolled it out incrementally rather than as a big-bang redesign. Documented live with an interactive demo built in.
